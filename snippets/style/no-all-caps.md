# No All-Caps Filenames

Use conventional file naming. Only README.md should be in all caps. All other documentation files should use lowercase with hyphens (kebab-case) such as:

- `installation-guide.md` (not INSTALLATION.md)
- `contributing.md` (not CONTRIBUTING.md)
- `changelog.md` (not CHANGELOG.md)
- `license.md` (not LICENSE.md, though LICENSE without extension is acceptable)

Follow language-specific conventions for code files:
- Python: `snake_case` (e.g., `data_processor.py`)
- JavaScript/TypeScript: `camelCase` or `kebab-case` (e.g., `dataProcessor.js` or `data-processor.js`)
- Java: `PascalCase` for classes (e.g., `DataProcessor.java`)
- Go: `lowercase` (e.g., `dataprocessor.go`)

All-caps filenames create visual clutter and imply emphasis where none is needed.
